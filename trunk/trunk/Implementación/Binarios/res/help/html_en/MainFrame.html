Main window actions

This window explains the application's main features in a general sense. You can browse each operation's corresponding page for a more detailed explanation.

Administrator actions



Manage case types

A case type is a Problem-solution pair in which concrete cases will be stored. The option Manage Case Types allows you to define, modify and delete case types, determining its attribute composition, the manner in which they're evaluated and how a query on that type is performed in each CBR step.

Manage users

This window allows you to create, modify and delete the different users of the application.

User actions

Execute CBR cycle

Executing a CBR (Case Based Reasoning) cycle consists of performing a query for a concrete case type. The user will be asked for the query data and a solution will be offered, which can then be evaluated from 0 to 100 for future queries.
The configured CBR query option is restricted to advanced users and allows to establish the metric (Evaluation method) for each attribute, its weight when searching for a solution and the methods to use in each CBR cycle step.

Import cases

The import cases option allows you to introduce cases for a concrete case type manually or from a WEKA file, so the case base of that type is more complete.

Statistics

Full stats of the application usage by its users. Regular users can only see their own stats, while administrators can see global stats or filter by user and case type
