Create a new case type

Case type components

A case type is composed of the following:
-A set of problem attributes. Each attribute has a weight and a default evaluation metric. which will be used in a query if the user doesn't/can't specify a value for those fields.
-A set ofsolution attributes.
-A list of method allowed in each cycle step. Out of those there will be one which will be used as the default in a basic cycle or if the user doesn't pick one while performing a query.(See help section about queries).
-The configuration of each allowed method.

Add/delete attributes



To add an attribute to the problem or solution, click the '+' button:. To delete it, click the button: . Please note there must be an attribute for the problem and one for the solution.

Fill out attributes

Its mandatory to fill out every field from each attribute. To do that, fill the name of each attribute (It must be unique). After that, you must pick the attribute type. The allowed types are:
-Character string For example:"Test"
-Integer number. For example:2
-Decimal number.For example:2.3
Finally, you must pick a default metric. The default metrics depend on the attribute type. For example, the longest substring comparison isn't available for Integer numbers or decimals. Some metrics require a configurating parameter. (See the metrics section in the basic concepts page)

Setting methods



To set the methods, click the 'Next' (Arrow) button. The program will check the following:
-That each attribute has been filled out and that there are no repeated attribute names.
-That the case type name has not been picked yet and doesn't contain spaces or special characters.
Then the methods panel will be shown, including a subpanel for each CBR cycle step. To the left of each method is a checkbox which indicates if that method will be available for the users (At least one will always be picked). The dropdown list of each panel allows you to pick which method will be used if the user can't/won't pick one when performing a query.

Some techniques can be configured. To do that, click the "Configure" button to the side of each method. If you choose not to configure it, it will take default values.

Last steps

To finish, click the Save button to save the case type, which from now on can be viewed and modified. Now you will also be able to perform all the other actions related to case types with the new case.
