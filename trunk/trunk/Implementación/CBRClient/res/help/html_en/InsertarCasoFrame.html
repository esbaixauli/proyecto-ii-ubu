Importing cases

The application allows to import cases of a defined type, filling the application's case base with them. This ensures that there exists enough data for the users to perform CBR queries, so the case base will get feedback with the information they provide.

File importing

You can import WEKA files in *.arff format (Check this link to see those files' format). The file should contain a column for each field of the case type that is being imported, whose name and data type match the Case Type specification. . Asimismo, debe haber un atributo de nombre META_QUALITY y tipo NUMERIC, que indica la medida de calidad del caso de 0 a 100. Con todo, un fichero *.arff válido debería tener el siguiente aspecto:



Para cargar el fichero, desde el panel de importar desde fichero, Pulse el botón "Abrir" y, con el fichero seleccionado (Su ruta aparecerá en el cuadro de texto a su izquierda) pulse "Cargar desde Fichero".


Verá como los casos aparecen en la tabla inferior. Los atributos del problema aparecen en rojo, los de la solución en azul y la calidad en verde.
Ahora, puede modificar, ordenar (pulsando la cabecera de cada columna) o buscar en la tabla (con el filtro que se muestra a continuación). Cuando haya terminado, puede pulsar el botón "Insertar casos" para completar la operación.


Recuerde que el filtro acepta expresiones regulares básicas, según la convención corriente.

Importación a mano

También se permite introducir manualmente un caso desde el panel inferior de Inserción manual. Simplemente rellénese el formulario y pulse el botón "Inserción Manual". Recuerde que la calidad es un atributo de 0 a 100.
