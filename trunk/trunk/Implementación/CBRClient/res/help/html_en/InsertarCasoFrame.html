Importing cases

The application allows to import cases of a defined type, filling the application's case base with them. This ensures that there exists enough data for the users to perform CBR queries, so the case base will get feedback with the information they provide.

File importing

You can import WEKA files in *.arff format (Check this link to see those files' format). The file should contain a column for each field of the case type that is being imported, whose name and data type match the Case Type specification. There must also exist an attribute named META_QUALITY of NUMERIC type, which indicates the quality level of the case from 0 to 100. To load the file, from the import from file panel, click on the "Open" button and once a file has been selected (Its path will appear on the text area to the left), click "Load from file".

Manual import

You can also introduce a new case from the manual import panel located at the lowermost of the window. Simply fill up the form and push the "Manual Insert" button. Remember that the quality is an attribute from 0 to 100.
