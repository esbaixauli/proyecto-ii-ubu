Performing a CBR query

Basic and configurated queries

Basic users can only perform basic queries. This kind of queries allow only to insert the problem data to obtain a solution. advanced users can perform configured queries. In them you can introduce, besides the problem data, the way (metric) and weight (degree of relevance) each attribute will be configured, as well as the technique that will be used in each step of the CBR cycle.

Performing a basic CBR query


To perform a basic query, simply introduce the problem's data. To the side of each attribute the data type to which it belongs is shown. If you try to introduce a value not allowed by the type you will be notified of it. To perform a query, click the Execute Cycle button if you wish to execute the complete cycle once or the Step by step if you want to see the intermediate results of each step in the cycle.

Perfoming a configured CBR query


Configuring the problem data

In a configured query, near each of the attributes will appear its weight (Relative importance against the other attributes) and its metric (Way in which the attribute will be compared). These can be modified by the user altering the values of the respective textfield and dropdown list.

Configuring methods

You are allowed to choose which method will be used in each CBR cycle step between many(determined by the administrator). To choose utilizing a concrete method, select it in its corresponding dropdown list. If you want a method that is not available for this problem, contact your administrator to have it enabled.

Once the method's been chosen, configure it clicking on the "Configure" button. For more info about each concrete method and how to configure it, browse the corresponding page in this manual.
