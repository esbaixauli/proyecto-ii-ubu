Case types

Case types represent the many kinds of problems the application can solve. Each type is composed of a set of problem attributes, a set of solution attributes and a measurement of the solution's quality, from 0 to 100.

Case types vs. cases

A case type represents a kind of problem and its corresponding solution type, while a case is a concrete occurrence of a case type.
For example, a case type "Car malfunction" may comprise the problem attributes:
"Malfunction Description"
"Engine temperature"
"Tyres pressure"
and its solution may comprise the attributes:
"Diagnose"
Meanwhile, a concrete case for this type could be:
"Smoke coming out of the hood","248ºF","33PSI"/"Engine overheating","3.5"

Actions over the case types

The actions which can be performed over a case type are:
Visualize: It allows to see the structure, metrics and methods of a case type, as well as modifying some of its parameters.
Create new: It creates a new case type, with the attributes and methods that compose it.
Erase: Erases the case type and unbinds all its users from it. All occurences of the case type will be erased, including its statistics and cases belonging to this type.
